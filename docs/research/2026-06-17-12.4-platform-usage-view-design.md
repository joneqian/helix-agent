# 12.4 平台用量视图 — 按租户 + agent 看 token 与成本 + rollup enable 配置入口

## 1. 需求(用户拍板)

平台(system_admin)能查看:
- **每个租户**的 token 消耗与成本
- **每个 agent** 的 token 消耗与成本

数据源 = `tenant_billing_ledger`(token + 成本一起,口径一致)。rollup 调度走 k8s cron(方向 A),
但 app 侧要有 **enable 开关配置入口**(cron 跑前读,关 → 跳过)。

排除:发票/对账单、支付(Stripe)、预算告警、cron 表达式(归 k8s)、状态查看 / 手动触发 / 参数化
(用户只要 enable 开关)。

## 2. 现状(核代码,非照搬 assessment)

计费链路已建 ~60%,**计量→定价→rollup→ledger→暴露** 全交付 + 接线 + 测试(Stream Y/Z):
- G.9 `token_usage`(实时,中间件每次 LLM 调用落库,含 agent_name/model/provider)
- Y3 `model_rate_card`(费率卡,micro-USD + markup_bps + tier + 时间版本,admin API+UI 全 CRUD)
- Y4 `BillingRollupJob` → `tenant_billing_ledger`,按 **(tenant, month, provider, model, agent_name)** 分桶,
  每桶带 token(input/output/cache)**和**成本三段(base/markup/billed micro-USD)
- Z chargeback:`GET /v1/admin/billing/chargeback`(system_admin,跨租户,全拆分)

**真 gap(已 grep 确认)**:
1. chargeback 端点(`api/billing_admin.py:79-94`)**只按 tenant_id 聚合,把 agent_name collapse 掉了** ——
   per-agent 数据在 ledger 有,admin API 没暴露。
2. rollup enable 无平台配置入口:job 只读 env(`HELIX_BILLING_ROLLUP_*`),admin 改不了;且 k8s cron 未接。

## 3. 设计

### Part A — per-agent 平台视图(token + 成本)

**API**:扩 `GET /v1/admin/billing/chargeback?month=&tenant_id=` —— 给 `tenant_id` 时响应**追加** per-agent 拆分:
```
data: {
  month, as_of,
  total_base_cost_micros, total_billed_cost_micros, total_margin_micros,
  tenants: [ {tenant_id, tokens, base/markup/billed/margin, unpriced_buckets} ],   # 现状不变
  agents: [ {agent_name, input/output/cache tokens, base/markup/billed/margin_micros, unpriced_buckets} ]  # 新增,仅 tenant_id 给定时
}
```
- 源 = `ledger.list_for_month_all_tenants(month)`(已读),过滤到 tenant_id 后按 **agent_name** 二次聚合;
  **零新计算 / 零新 store 方法**(ledger 桶已含 agent + token + 成本)。
- 向后兼容:不给 tenant_id → `agents` 省略(现状响应不变)。
- 鉴权不变:`require("billing","read")` + inline `is_system_admin`,`bypass_rls_session()`。

**admin-ui**:`SettingsBillingChargeback.tsx` 钻取 —— 选某租户(已有 tenant filter)→ 展开 per-agent 行表
(agent / token / base / markup / billed / margin)。SDK `billing-admin.ts` 加 `agents` 字段类型。

### Part B — rollup enable 配置入口(平台全局,cron 读)

镜像 `platform_judge_config` 单行模式(NULL-tenant,无 RLS,bypass_rls_session 访问):
- **表** `platform_billing_config`(0083 migration):`id="singleton"` PK / `rollup_enabled bool NOT NULL default true`
  / `updated_at` / `updated_by`。
- **store** `persistence/platform_billing_config/`(base/sql/memory):`get() -> Row | None` / `put(rollup_enabled, updated_by)`。
- **control-plane**:service 包装 + `api/platform_billing_config.py` `GET`/`PUT /v1/platform/billing-config`
  (system_admin + `billing:write`,审计 `PLATFORM_BILLING_CONFIG_UPDATED`)。
- **admin-ui**:`settings_platform/PlatformBillingSection.tsx`(enable 开关)挂进 `SettingsPlatformConfig.tsx` +
  `api/platform_billing_config.ts` SDK。
- **rollup job**:`main._amain` 跑前读 `DbPlatformBillingConfigStore.get()`;`rollup_enabled is False` → log 跳过不跑
  (absent row / True → 照跑,默认开向后兼容)。

## 4. 不做 / 边界

发票/支付/预算告警(M2 / 外部);cron 表达式(k8s);rollup 状态查看 / 手动触发 / 参数化(用户只要 enable)。
成本必须走 ledger(token_usage 无价),故 per-agent 成本有 rollup-lag —— 由 cron 频率决定,非本设计范畴。

## 5. 测试

- Part A:chargeback 给 tenant_id → per-agent 聚合正确 / 不给 → agents 省略向后兼容 / system_admin 闸 /
  跨租户 bypass / no-leak(per-agent 成本只在 admin 面,租户面 `/v1/usage` 不变)。
- Part B:store get/put 单行幂等(memory + 真 PG)/ API system_admin 闸 + 审计 / rollup job disabled 跳过
  (job 级测:flag False → run_once 不调用)/ admin-ui section toggle。

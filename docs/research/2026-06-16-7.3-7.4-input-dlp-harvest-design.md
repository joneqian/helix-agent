## 1. 7.3 — 结构化不可信输入通道（PI-1c）

### 1.1 设计

业务系统把"待处理数据"（工单/邮件/文档）从 `input` 里**拆出来结构化传**，helix 知道"哪段不可信"
→ 复用既有 `spotlight_untrusted` 包裹。这是治内联注入的**根**（OWASP/StruQ 同源洞见：LLM 输入无
指令/数据分隔正是注入根因）。

- **API**：`RunRequest` 增 `untrusted_content: list[str] | None`（bounded：≤16 段、每段 ≤8192）。
  不传 → 同今天（纯 `input`，由 7.4 出站筛查兜底）。**向后兼容**。
- **nonce 一致性**：spotlight 的 per-build nonce 现活在 orchestrator 图闭包（`agent_factory.py:677`），
  控制面 seed 时拿不到。**把 nonce 从 `BuiltAgent` 暴露出来**（`BuiltAgent.spotlight_nonce: str | None`），
  控制面 seed 用**同一 nonce** 包裹 untrusted 段 → 与图内 tool/RAG 包裹同源，且系统提示 clause
  （`SPOTLIGHT_SYSTEM_CLAUSE`，build 时已注入）解释的是 marker **格式**（`«UNTRUSTED nonce=…»`），
  覆盖任意 nonce 值。
- **接入点**：`_build_human_message` 增 `untrusted_content` + `spotlight_nonce` 入参，在文本主体后
  追加每段 `spotlight_untrusted(block, nonce=…)`。`spotlight_nonce` 为 None（agent 关了 spotlight）→
  **不包裹**（degrade：原样附加，与今天行为一致，靠 7.4 兜）。多模态 path A（content blocks）→ 追加
  text block；plain path → 追加文本。

### 1.2 验收（live E2E，CI green ≠ live works）

红队 `injection-001/002/003`（inline 注入）原本 3/3 泄漏。把每条的"文档/工单"段走 `untrusted_content`
结构化传 → 真 deepseek agent 重跑 → canary **不泄漏**（SAFE）。这是 7.3 ★5 的硬门（非纯单测）。

## 2. 7.4 — 出站 DLP（条件输出）

### 2.1 设计

model **响应**回到用户/下游工具前，分类其中 PII 类别并**脱敏**（"条件输出"=按分类 redact，区别于
output_screen 的 block）。

- **模块**：`common/dlp.py`（helix-common，与 `output_screen.py` 同层）。自含 PII 类别 regex
  （email / 手机 CN / 身份证 CN / 信用卡），与 `audit/redactor.py:PII_PATTERNS` 同源思路——**不跨层
  import**（helix-common 不依赖 helix-runtime），4 条 regex 自含 + 注释标注关系。
  - `scan_and_redact(text) -> DlpResult(redacted: str, categories: tuple[str,...])`：纯+确定性，
    matched 值**绝不**返回/记日志（只回类别名，避免二次泄漏，CodeQL py/clear-text-logging）。
- **manifest**：`DefenseSpec.output_dlp: Literal["redact", "off"] = "off"`。**默认 off**——出站脱敏会
  改动**合法**输出（agent 正当回答"你的邮箱是…"会被打码），与 `output_judge`/`action_screen` 同为
  **opt-in 防御**（house pattern：会改行为的防御默认关，高敏 agent 显式开）。能力本身完整（≠弱能力
  包装成设计选择）：开启即全类别分类+脱敏+指标+审计。
- **接入点**：`builder.py` 输出路径，在 `output_screen` block 检查**之后**、仅对**终态**响应
  （无 tool_calls，与 judge 同位）跑。block 命中则跳过（已拦）。脱敏后 `model_copy(content=…)`。
- **可观测**：指标 `helix_output_dlp_redacted_total{category}` + 审计 action（命中类别，非值）。

### 2.2 验收

确定性 redaction → 单测（各类别命中+脱敏 / off→原样 / matched 值不外泄）+ 装配测（manifest
开关 → builder 接线生效）。与 3.3 context-pressure 同档：确定性，单测+装配测充分，无需 live E2E。

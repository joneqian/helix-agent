# Dev mTLS materials

Generated by ``tools/dev-certs/generate.py`` into ``infra/dev-certs/``.
The directory is git-ignored — the private keys must never be committed.

## Bootstrap

```bash
uv run python tools/dev-certs/generate.py
```

This writes ``ca.crt`` + per-service client/server PEMs under
``infra/dev-certs/``. The dev stack mounts the directory into the nginx
sidecar (see ``infra/docker-compose.yml``).

## Identities issued

| File pair | Purpose |
|-----------|---------|
| ``ca.{crt,key}`` | Self-signed root CA (5y) |
| ``server.{crt,key}`` | nginx TLS server cert (1y) |
| ``orchestrator.{crt,key}`` | Stream E orchestrator → control plane |
| ``sandbox-supervisor.{crt,key}`` | Stream F sandbox supervisor → control plane |
| ``control-plane.{crt,key}`` | Loopback identity used by integration tests |

CN values match ``settings.mtls_allowed_service_subjects`` on the
control plane. Add a new entry there + to ``SERVICE_IDENTITIES`` in
``generate.py`` whenever a new service joins the mesh.

## Rotation

Re-run ``generate.py --force`` to mint a fresh PKI. Production rotation
is managed by cert-manager and is out of scope for M0 (per ADR-0003).
